Session-Centric Design — v3

CHAT_SESSION remains the root. The key additions are SESSION_LEG (tracks every participant segment within a session) and TRANSFER_EVENT (full audit of every handoff). All IDs changed from UUID to BIGINT for simpler queries and better join performance.

🏛️ Full Table Hierarchy
CHAT_SESSION ← ROOT · session_id BIGINT · no outward FK
│
├──MOBILE_DEVICE· captured at start · 1:1
├──UCID_MAPPING· async, inserted later · 1:1
├──DISCONNECT_EVENT· written at session end · 1:1
├──LICENSE_EVENT· session_id nullable, independent audit · 1:N
└──SESSION_LEG· one row per participant segment · 1:N
│ ↺ prev_leg_id (self-ref) → transfer chain
├──TRANSFER_EVENT· from_leg + to_leg audit · 1:N
└──CHAT_MESSAGE· messages now link to LEG · 1:N
📐 Key Design Decisions
DECISIONWHATWHY
ID TYPE
BIGINT not UUID		All PKs/FKs are BIGINT IDENTITY auto-incrementSimpler joins, smaller index size, faster range queries. No need for distributed generation in a single-system chat platform.
NEW TABLE
SESSION_LEG		One row per participant segment — each Agent, Bot, Supervisor, Customer active periodWithout this table there's no way to know which agent handled which part of the session, or reconstruct the transfer path. Replaces storing participant info on the session itself.
SELF-REF FK
prev_leg_id		SESSION_LEG.prev_leg_id is a FK back to SESSION_LEG itselfCreates a linked-list chain of legs. Leg 1 → Leg 2 (prev=1) → Leg 3 (prev=2). You can walk the full transfer path for any session by following prev_leg_id.
NEW TABLE
TRANSFER_EVENT		Dedicated transfer audit table. from_leg_id + to_leg_id + transfer_type + statusprev_leg_id alone can tell you the chain, but not the timing, reason, who initiated it, or whether it failed. TRANSFER_EVENT captures all that independently.
MSG CHANGE
Messages link to LEG		CHAT_MESSAGE now has both session_id and leg_id FKsWithout leg_id you can't tell which agent wrote which message. With leg_id you can instantly filter "all messages from Agent 47 in this session" or "all messages before the transfer".
PARTICIPANT IDs
Separate ID columns per role		customer_id, agent_id, supervisor_id, bot_id — only one is non-NULL per leg rowEach role has its own ID space. One column per role makes it easy to index and query per role type. participant_type tells you which column is populated.

Table Deep-Dive

Click any table to expand field-by-field explanations, including design notes for the new SESSION_LEG and TRANSFER_EVENT tables.

🦵 SESSION_LEG NEW · 1:N · CORE ROUTING ▼

The most important new table. One row is created for every participant segment in a session. The customer's leg is created at session start. Each time a new agent, bot, or supervisor joins, a new leg row is created. prev_leg_id is a self-referencing FK that chains all legs together into a transfer path — like a linked list.

PK leg_id BIGINT
Auto-generated identity. Unique across all legs in the system.
FK session_id BIGINT
Links this leg to its parent session. All legs for one session share the same session_id.
leg_sequence INT
1 = first leg, 2 = second, etc. Unique per session. Tells you the order legs occurred without parsing timestamps.
participant_type ENUM
customer / agent / supervisor / bot. Tells you which ID column is populated and what role this participant has.
customer_id / agent_id / supervisor_id / bot_id BIGINT nullable
Only one is non-NULL per row, matching participant_type. These are the external IDs from your identity systems for each role.
FK↺ prev_leg_id BIGINT nullable
Self-referencing FK. NULL for the first leg. Points to the leg that was active before this one. This creates the transfer chain: follow prev_leg_id backwards to see every handoff.
leg_status ENUM
active (currently handling) / completed (ended cleanly) / transferred (handed off to next leg) / dropped (connection lost mid-leg).
Design note — why separate ID columns instead of one participant_id? Each role (customer, agent, supervisor, bot) has its own ID space and the numbers could overlap. customer_id=5 and agent_id=5 are different people. Keeping them in separate columns makes this unambiguous and enables per-role indexes.
🔄 TRANSFER_EVENT NEW · FULL TRANSFER AUDIT ▼

A dedicated audit table that records every transfer that happened in a session. While prev_leg_id on SESSION_LEG shows the chain, TRANSFER_EVENT captures the full details: who initiated it, what type it was, when it completed, and whether it succeeded or failed.

FK from_leg_id BIGINT
The leg that was transferring away. Points to the SESSION_LEG row of the departing participant.
FK to_leg_id BIGINT nullable
NULL until the receiving participant's leg is created. Updated once the new leg row exists. NULL state captures the "transfer in progress" window.
transfer_type ENUM
agent_to_agent / agent_to_supervisor / bot_to_agent / bot_to_supervisor / customer_to_agent / agent_to_bot. Allows filtering all escalations, all bot handoffs, etc.
initiated_by ENUM
Who triggered the transfer: customer / agent / supervisor / bot / system. Important for analytics — was this a customer request or an agent escalation?
status ENUM
pending (initiated, waiting for receiver) / completed (receiver picked up) / failed (transfer rejected or timed out).
Design note: prev_leg_id and TRANSFER_EVENT are complementary. prev_leg_id is the fast structural link (great for JOINs and chain traversal). TRANSFER_EVENT is the rich audit record (timing, reason, who initiated, success/failure). You need both.
✉️ CHAT_MESSAGE UPDATED — now links to LEG ▼

Messages now carry both session_id and leg_id. session_id lets you pull all messages for a session quickly. leg_id tells you exactly which agent/bot/supervisor was active when each message was sent — critical for quality review, agent performance metrics, and post-transfer conversation analysis.

FK session_id BIGINT
Still present for fast "get all messages in session" queries without needing to JOIN through leg.
FK leg_id BIGINT
NEW. Links to the SESSION_LEG row — tells you which participant segment this message belongs to.
sender_id BIGINT
The specific ID of the sender (customer_id / agent_id / supervisor_id / bot_id). Together with sender_type, fully identifies who sent each message.
sequence_no INT
Scoped per session. Guarantees conversation order even if timestamps are identical or out of order.

Use Case Scenarios

How the new SESSION_LEG and TRANSFER_EVENT tables handle real routing situations.

✅ Customer → Bot → Agent COMMON

Customer connects, bot handles first, then transfers to human agent.

  • →CHAT_SESSION created, status=active
  • →LEG #1: type=customer, customer_id=C1, prev=NULL
  • →LEG #2: type=bot, bot_id=B1, prev=Leg1
  • →Messages (Leg2): customer + bot messages, leg_id=Leg2
  • →TRANSFER_EVENT: bot_to_agent, from=Leg2, to=NULL (pending)
  • →LEG #3: type=agent, agent_id=A1, prev=Leg2
  • →TRANSFER_EVENT.to_leg_id updated to Leg3, status=completed
  • →Messages (Leg3): agent messages, leg_id=Leg3
⬆️ Agent Escalates to Supervisor ESCALATION

Agent can't resolve, escalates to supervisor who takes over.

  • →LEG #1: type=customer, customer_id=C1, prev=NULL
  • →LEG #2: type=agent, agent_id=A1, prev=Leg1, status=active
  • →Agent initiates transfer → TRANSFER_EVENT: agent_to_supervisor
  • →LEG #3: type=supervisor, supervisor_id=S1, prev=Leg2
  • →Leg2 status=transferred, Leg3 status=active
  • →Messages from supervisor have leg_id=Leg3
  • →Session ends: DISCONNECT_EVENT, Leg3 status=completed
🔁 Agent → Agent Transfer HANDOFF

First agent transfers to specialist agent. Chain is preserved.

  • →LEG #1: type=customer, customer_id=C1
  • →LEG #2: type=agent, agent_id=A1 (general support)
  • →TRANSFER_EVENT: agent_to_agent, initiated_by=agent
  • →LEG #3: type=agent, agent_id=A2 (specialist), prev=Leg2
  • →Full chain queryable: Leg3 → prev → Leg2 → prev → Leg1
  • →Messages before transfer: leg_id=Leg2. After: leg_id=Leg3
🔴 tcomm License Kill EDGE CASE

License limit fires, tcomm kills session mid-leg.

  • →LICENSE_EVENT inserted (possibly session_id=NULL)
  • →Active leg status=dropped
  • →DISCONNECT_EVENT: is_license_limit=TRUE, tcomm_triggered=TRUE
  • →CHAT_SESSION status=license_exceeded
  • →All messages already saved with leg_id — no data lost
  • →Transfer chain intact even though session ended abruptly
🔍 Query: Full Session Reconstruction ANALYTICS

Reconstruct who handled what, when, and what was said.

  • →JOIN CHAT_SESSION + SESSION_LEG on session_id ORDER BY leg_sequence
  • →For each leg: see participant_type, their ID, start/end times
  • →JOIN TRANSFER_EVENT to see why/how each transfer happened
  • →JOIN CHAT_MESSAGE with leg_id to get messages per participant
  • →JOIN UCID_MAPPING to tie back to telephony record
  • →Full session timeline reconstructable from DB alone
🤖 Failed Transfer FAILURE

Transfer initiated but receiving agent never picks up.

  • →TRANSFER_EVENT inserted: status=pending, to_leg_id=NULL
  • →Timeout fires — no agent accepted
  • →TRANSFER_EVENT.status=failed, completed_at=timeout time
  • →Original leg may resume or session drops
  • →DISCONNECT_EVENT records final outcome
  • →Failed transfers are queryable: WHERE status='failed'

💬CHAT_SESSIONROOT · no FK outwardGenerated first. All others reference this.
CREATE TABLE chat_session (
    session_id   BIGINT       NOT NULL GENERATED ALWAYS AS IDENTITY,
    start_time   TIMESTAMP    NOT NULL DEFAULT NOW(),
    end_time     TIMESTAMP,
    status       VARCHAR(32)  NOT NULL DEFAULT 'active'
                              CHECK (status IN ('active','ended','dropped','license_exceeded')),
    channel      VARCHAR(32),
    created_at   TIMESTAMP    NOT NULL DEFAULT NOW(),
    metadata     JSONB,
    CONSTRAINT pk_chat_session PRIMARY KEY (session_id)
);
CREATE INDEX idx_cs_status  ON chat_session (status);
CREATE INDEX idx_cs_created ON chat_session (created_at DESC);

    os_version         VARCHAR(32),

CREATE INDEX idx_md_session   ON mobile_device (session_id);

🦵SESSION_LEG⭐ NEW · 1:N · COREOne row per participant segment. prev_leg_id = transfer chain.
CREATE TABLE session_leg (
    leg_id            BIGINT       NOT NULL GENERATED ALWAYS AS IDENTITY,
    session_id        BIGINT       NOT NULL,
    leg_sequence      INT          NOT NULL,          -- order within session
    participant_type  VARCHAR(16)  NOT NULL
                                   CHECK (participant_type IN ('customer','agent','supervisor','bot')),
    -- only one non-NULL per row (matches participant_type)
    customer_id       BIGINT,
    agent_id          BIGINT,
    supervisor_id     BIGINT,
    bot_id            BIGINT,
    -- transfer chain (self-referencing FK)
    prev_leg_id       BIGINT,                         -- NULL for first leg
    transfer_reason   VARCHAR(128),
    leg_start         TIMESTAMP    NOT NULL DEFAULT NOW(),
    leg_end           TIMESTAMP,
    leg_status        VARCHAR(32)  NOT NULL DEFAULT 'active'
                                   CHECK (leg_status IN ('active','completed','transferred','dropped')),
    CONSTRAINT pk_session_leg  PRIMARY KEY (leg_id),
    CONSTRAINT fk_sl_session   FOREIGN KEY (session_id)  REFERENCES chat_session(session_id) ON DELETE CASCADE,
    CONSTRAINT fk_sl_prev_leg  FOREIGN KEY (prev_leg_id) REFERENCES session_leg(leg_id),
    CONSTRAINT uq_sl_sequence  UNIQUE (session_id, leg_sequence)
);
CREATE INDEX idx_sl_session    ON session_leg (session_id);
CREATE INDEX idx_sl_customer   ON session_leg (customer_id)   WHERE customer_id   IS NOT NULL;
CREATE INDEX idx_sl_agent      ON session_leg (agent_id)      WHERE agent_id      IS NOT NULL;
CREATE INDEX idx_sl_supervisor ON session_leg (supervisor_id) WHERE supervisor_id IS NOT NULL;
CREATE INDEX idx_sl_bot        ON session_leg (bot_id)        WHERE bot_id        IS NOT NULL;
CREATE INDEX idx_sl_prev_leg   ON session_leg (prev_leg_id)   WHERE prev_leg_id   IS NOT NULL;
🔄TRANSFER_EVENT⭐ NEW · AUDITFull details of every transfer — type, reason, initiator, status.
CREATE TABLE transfer_event (
    transfer_id       BIGINT       NOT NULL GENERATED ALWAYS AS IDENTITY,
    session_id        BIGINT       NOT NULL,
    from_leg_id       BIGINT       NOT NULL,
    to_leg_id         BIGINT,                         -- NULL until receiving leg created
    transfer_type     VARCHAR(32)  NOT NULL
                                   CHECK (transfer_type IN (
                                     'agent_to_agent','agent_to_supervisor',
                                     'bot_to_agent','bot_to_supervisor',
                                     'customer_to_agent','agent_to_bot')),
    transfer_reason   VARCHAR(128),
    initiated_by      VARCHAR(16)  CHECK (initiated_by IN ('customer','agent','supervisor','bot','system')),
    transferred_at    TIMESTAMP    NOT NULL DEFAULT NOW(),
    completed_at      TIMESTAMP,
    status            VARCHAR(32)  NOT NULL DEFAULT 'pending'
                                   CHECK (status IN ('pending','completed','failed')),
    CONSTRAINT pk_transfer_event PRIMARY KEY (transfer_id),
    CONSTRAINT fk_te_session     FOREIGN KEY (session_id)  REFERENCES chat_session(session_id) ON DELETE CASCADE,
    CONSTRAINT fk_te_from_leg    FOREIGN KEY (from_leg_id) REFERENCES session_leg(leg_id),
    CONSTRAINT fk_te_to_leg      FOREIGN KEY (to_leg_id)   REFERENCES session_leg(leg_id)
);
CREATE INDEX idx_te_session  ON transfer_event (session_id);
CREATE INDEX idx_te_from_leg ON transfer_event (from_leg_id);
CREATE INDEX idx_te_to_leg   ON transfer_event (to_leg_id) WHERE to_leg_id IS NOT NULL;

🔍 Common Queries — PostgreSQL
Full session with all legs and transfer chain
SELECT
  s.session_id, s.status, s.start_time,
  sl.leg_id, sl.leg_sequence, sl.participant_type,
  sl.agent_id, sl.bot_id, sl.supervisor_id,
  sl.leg_status, sl.prev_leg_id
FROM chat_session s
JOIN session_leg sl USING (session_id)
WHERE s.session_id = 12345
ORDER BY sl.leg_sequence ASC;
Messages grouped by leg (who said what, when)
SELECT
  sl.leg_sequence, sl.participant_type,
  cm.sequence_no, cm.sender_type, cm.content, cm.sent_at
FROM chat_message cm
JOIN session_leg sl USING (leg_id)
WHERE cm.session_id = 12345
ORDER BY cm.sequence_no ASC;
All transfers for a session with timing
SELECT
  te.transfer_type, te.initiated_by, te.status,
  te.transferred_at, te.completed_at,
  fl.participant_type AS from_type,
  tl.participant_type AS to_type
FROM transfer_event te
JOIN session_leg fl ON te.from_leg_id = fl.leg_id
LEFT JOIN session_leg tl ON te.to_leg_id = tl.leg_id
WHERE te.session_id = 12345;
All sessions handled by a specific agent
SELECT DISTINCT s.session_id, s.start_time, s.status
FROM session_leg sl
JOIN chat_session s USING (session_id)
WHERE sl.agent_id = 99
ORDER BY s.start_time DESC;
All tcomm license kills
SELECT s.session_id, s.start_time, d.disconnected_at, d.reason_detail
FROM disconnect_event d
JOIN chat_session s USING (session_id)
WHERE d.is_license_limit = TRUE AND d.tcomm_triggered = TRUE
ORDER BY d.disconnected_at DESC;

USE ChatSessionDB;
GO

-- ── TABLE 1: CHAT_SESSION  (ROOT)
CREATE TABLE chat_session (
    session_id   BIGINT       NOT NULL IDENTITY(1,1),
    start_time   DATETIME2    NOT NULL DEFAULT SYSDATETIME(),
    end_time     DATETIME2    NULL,
    status       NVARCHAR(32) NOT NULL DEFAULT 'active'
                              CONSTRAINT chk_cs_status CHECK (
                                status IN ('active','ended','dropped','license_exceeded')),
    channel      NVARCHAR(32) NULL,
    created_at   DATETIME2    NOT NULL DEFAULT SYSDATETIME(),
    metadata     NVARCHAR(MAX) NULL,
    CONSTRAINT PK_chat_session PRIMARY KEY CLUSTERED (session_id)
);
GO
CREATE NONCLUSTERED INDEX IDX_cs_status  ON chat_session (status);
CREATE NONCLUSTERED INDEX IDX_cs_created ON chat_session (created_at DESC);
GO

-- ── TABLE 2: MOBILE_DEVICE
CREATE TABLE mobile_device (
    device_id          BIGINT       NOT NULL IDENTITY(1,1),
    session_id         BIGINT       NOT NULL,
    mobile_id          NVARCHAR(64) NOT NULL,
    platform           NVARCHAR(32) NULL,
    os_version         NVARCHAR(32) NULL,
    app_version        NVARCHAR(32) NULL,
    device_fingerprint NVARCHAR(128) NULL,
    captured_at        DATETIME2    NOT NULL DEFAULT SYSDATETIME(),
    CONSTRAINT PK_mobile_device    PRIMARY KEY CLUSTERED (device_id),
    CONSTRAINT FK_md_session       FOREIGN KEY (session_id) REFERENCES chat_session(session_id) ON DELETE CASCADE,
    CONSTRAINT UQ_mobile_device_id UNIQUE (mobile_id)
);
GO
CREATE NONCLUSTERED INDEX IDX_md_session   ON mobile_device (session_id);
CREATE NONCLUSTERED INDEX IDX_md_mobile_id ON mobile_device (mobile_id);
GO

-- ── TABLE 3: UCID_MAPPING
CREATE TABLE ucid_mapping (
    ucid_map_id       BIGINT        NOT NULL IDENTITY(1,1),
    session_id        BIGINT        NOT NULL,
    ucid_value        NVARCHAR(128) NOT NULL,
    mobile_id         NVARCHAR(64)  NULL,
    resolved_at       DATETIME2     NOT NULL DEFAULT SYSDATETIME(),
    source_system     NVARCHAR(64)  NULL,
    resolution_status NVARCHAR(32)  NOT NULL DEFAULT 'resolved'
                                    CONSTRAINT chk_um_status CHECK (
                                      resolution_status IN ('resolved','failed','pending')),
    notes             NVARCHAR(MAX) NULL,
    CONSTRAINT PK_ucid_mapping PRIMARY KEY CLUSTERED (ucid_map_id),
    CONSTRAINT FK_um_session   FOREIGN KEY (session_id) REFERENCES chat_session(session_id) ON DELETE CASCADE,
    CONSTRAINT UQ_ucid_value   UNIQUE (ucid_value)
);
GO
CREATE NONCLUSTERED INDEX IDX_um_session ON ucid_mapping (session_id);
CREATE NONCLUSTERED INDEX IDX_um_value   ON ucid_mapping (ucid_value);
GO

-- ── TABLE 4: SESSION_LEG  (1:N — core routing table)
--    prev_leg_id is a self-referencing FK — tracks transfer chain.
CREATE TABLE session_leg (
    leg_id            BIGINT        NOT NULL IDENTITY(1,1),
    session_id        BIGINT        NOT NULL,
    leg_sequence      INT           NOT NULL,
    participant_type  NVARCHAR(16)  NOT NULL
                                    CONSTRAINT chk_sl_type CHECK (
                                      participant_type IN ('customer','agent','supervisor','bot')),
    customer_id       BIGINT        NULL,
    agent_id          BIGINT        NULL,
    supervisor_id     BIGINT        NULL,
    bot_id            BIGINT        NULL,
    prev_leg_id       BIGINT        NULL,
    transfer_reason   NVARCHAR(128) NULL,
    leg_start         DATETIME2     NOT NULL DEFAULT SYSDATETIME(),
    leg_end           DATETIME2     NULL,
    leg_status        NVARCHAR(32)  NOT NULL DEFAULT 'active'
                                    CONSTRAINT chk_sl_status CHECK (
                                      leg_status IN ('active','completed','transferred','dropped')),
    CONSTRAINT PK_session_leg    PRIMARY KEY CLUSTERED (leg_id),
    CONSTRAINT FK_sl_session     FOREIGN KEY (session_id)  REFERENCES chat_session(session_id) ON DELETE CASCADE,
    CONSTRAINT FK_sl_prev_leg    FOREIGN KEY (prev_leg_id) REFERENCES session_leg(leg_id),
    CONSTRAINT UQ_sl_sequence    UNIQUE (session_id, leg_sequence)
);
GO
CREATE NONCLUSTERED INDEX IDX_sl_session    ON session_leg (session_id);
CREATE NONCLUSTERED INDEX IDX_sl_customer   ON session_leg (customer_id)   WHERE customer_id   IS NOT NULL;
CREATE NONCLUSTERED INDEX IDX_sl_agent      ON session_leg (agent_id)      WHERE agent_id      IS NOT NULL;
CREATE NONCLUSTERED INDEX IDX_sl_supervisor ON session_leg (supervisor_id) WHERE supervisor_id IS NOT NULL;
CREATE NONCLUSTERED INDEX IDX_sl_bot        ON session_leg (bot_id)        WHERE bot_id        IS NOT NULL;
CREATE NONCLUSTERED INDEX IDX_sl_prev_leg   ON session_leg (prev_leg_id)   WHERE prev_leg_id   IS NOT NULL;
GO

-- ── TABLE 5: TRANSFER_EVENT  (full transfer audit)
CREATE TABLE transfer_event (
    transfer_id       BIGINT        NOT NULL IDENTITY(1,1),
    session_id        BIGINT        NOT NULL,
    from_leg_id       BIGINT        NOT NULL,
    to_leg_id         BIGINT        NULL,
    transfer_type     NVARCHAR(32)  NOT NULL
                                    CONSTRAINT chk_te_type CHECK (
                                      transfer_type IN (
                                        'agent_to_agent','agent_to_supervisor',
                                        'bot_to_agent','bot_to_supervisor',
                                        'customer_to_agent','agent_to_bot')),
    transfer_reason   NVARCHAR(128) NULL,
    initiated_by      NVARCHAR(16)  NULL
                                    CONSTRAINT chk_te_init CHECK (
                                      initiated_by IN ('customer','agent','supervisor','bot','system')),
    transferred_at    DATETIME2     NOT NULL DEFAULT SYSDATETIME(),
    completed_at      DATETIME2     NULL,
    status            NVARCHAR(32)  NOT NULL DEFAULT 'pending'
                                    CONSTRAINT chk_te_status CHECK (
                                      status IN ('pending','completed','failed')),
    CONSTRAINT PK_transfer_event PRIMARY KEY CLUSTERED (transfer_id),
    CONSTRAINT FK_te_session     FOREIGN KEY (session_id)  REFERENCES chat_session(session_id) ON DELETE CASCADE,
    CONSTRAINT FK_te_from_leg    FOREIGN KEY (from_leg_id) REFERENCES session_leg(leg_id),
    CONSTRAINT FK_te_to_leg      FOREIGN KEY (to_leg_id)   REFERENCES session_leg(leg_id)
);
GO
CREATE NONCLUSTERED INDEX IDX_te_session  ON transfer_event (session_id);
CREATE NONCLUSTERED INDEX IDX_te_from_leg ON transfer_event (from_leg_id);
CREATE NONCLUSTERED INDEX IDX_te_to_leg   ON transfer_event (to_leg_id) WHERE to_leg_id IS NOT NULL;
GO

-- ── TABLE 6: CHAT_MESSAGE  (1:N with session_leg)
CREATE TABLE chat_message (
    message_id   BIGINT        NOT NULL IDENTITY(1,1),
    session_id   BIGINT        NOT NULL,
    leg_id       BIGINT        NOT NULL,
    sequence_no  INT           NOT NULL,
    sender_type  NVARCHAR(16)  NOT NULL
                               CONSTRAINT chk_cm_sender CHECK (
                                 sender_type IN ('customer','agent','supervisor','bot')),
    sender_id    BIGINT        NULL,
    message_type NVARCHAR(32)  NULL,
    content      NVARCHAR(MAX) NULL,
    raw_payload  NVARCHAR(MAX) NULL,
    sent_at      DATETIME2     NULL,
    received_at  DATETIME2     NOT NULL DEFAULT SYSDATETIME(),
    CONSTRAINT PK_chat_message PRIMARY KEY CLUSTERED (message_id),
    CONSTRAINT FK_cm_session   FOREIGN KEY (session_id) REFERENCES chat_session(session_id) ON DELETE CASCADE,
    CONSTRAINT FK_cm_leg       FOREIGN KEY (leg_id)     REFERENCES session_leg(leg_id),
    CONSTRAINT UQ_cm_sequence  UNIQUE (session_id, sequence_no)
);
GO
CREATE NONCLUSTERED INDEX IDX_cm_session_seq ON chat_message (session_id, sequence_no ASC);
CREATE NONCLUSTERED INDEX IDX_cm_leg         ON chat_message (leg_id);
CREATE NONCLUSTERED INDEX IDX_cm_sender      ON chat_message (sender_id, sent_at DESC) WHERE sender_id IS NOT NULL;
GO

-- ── TABLE 7: DISCONNECT_EVENT
CREATE TABLE disconnect_event (
    disconnect_id    BIGINT        NOT NULL IDENTITY(1,1),
    session_id       BIGINT        NOT NULL,
    reason_code      NVARCHAR(64)  NULL,
    reason_detail    NVARCHAR(MAX) NULL,
    is_license_limit BIT           NOT NULL DEFAULT 0,
    tcomm_triggered  BIT           NOT NULL DEFAULT 0,
    error_code       NVARCHAR(32)  NULL,
    disconnected_at  DATETIME2     NOT NULL DEFAULT SYSDATETIME(),
    raw_payload      NVARCHAR(MAX) NULL,
    CONSTRAINT PK_disconnect_event PRIMARY KEY CLUSTERED (disconnect_id),
    CONSTRAINT FK_de_session       FOREIGN KEY (session_id) REFERENCES chat_session(session_id) ON DELETE CASCADE,
    CONSTRAINT UQ_de_session       UNIQUE (session_id)
);
GO
CREATE NONCLUSTERED INDEX IDX_de_license_limit ON disconnect_event (is_license_limit, disconnected_at DESC);
CREATE NONCLUSTERED INDEX IDX_de_tcomm         ON disconnect_event (tcomm_triggered,  disconnected_at DESC);
GO

-- ── TABLE 8: LICENSE_EVENT  (session_id nullable — independent audit)
CREATE TABLE license_event (
    license_event_id  BIGINT        NOT NULL IDENTITY(1,1),
    session_id        BIGINT        NULL,
    mobile_id         NVARCHAR(64)  NULL,
    event_type        NVARCHAR(64)  NULL,
    license_limit_hit BIT           NOT NULL DEFAULT 0,
    concurrent_count  INT           NULL,
    max_allowed       INT           NULL,
    triggered_at      DATETIME2     NOT NULL DEFAULT SYSDATETIME(),
    raw_payload       NVARCHAR(MAX) NULL,
    CONSTRAINT PK_license_event PRIMARY KEY CLUSTERED (license_event_id),
    CONSTRAINT FK_le_session    FOREIGN KEY (session_id) REFERENCES chat_session(session_id) ON DELETE SET NULL
);
GO
CREATE NONCLUSTERED INDEX IDX_le_limit_hit    ON license_event (license_limit_hit, triggered_at DESC);
CREATE NONCLUSTERED INDEX IDX_le_null_session ON license_event (triggered_at DESC) WHERE session_id IS NULL;
GO
🔍 Key Query Differences — T-SQL
T-SQL: use ON instead of USING, BIT = 1 for true
SELECT s.session_id, sl.leg_sequence, sl.participant_type, sl.agent_id
FROM chat_session s
JOIN session_leg sl ON s.session_id = sl.session_id
WHERE s.session_id = 12345
ORDER BY sl.leg_sequence ASC;

-- License kills: use 1 not TRUE for BIT columns
SELECT s.session_id, d.disconnected_at
FROM disconnect_event d
JOIN chat_session s ON s.session_id = d.session_id
WHERE d.is_license_limit = 1 AND d.tcomm_triggered = 1;
